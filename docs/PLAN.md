# 기획서 #6 — 사무실/교실 유틸 모음 (가칭: 다모아툴)

> 작성일: 2026-07-10 | 상태: Draft v1 | 배포: GitHub Pages 또는 Vercel 단독, 비용 0원

---

## 1. 문제 정의

- "사다리타기", "룰렛 돌리기", "팀 나누기", "자리 바꾸기", "타이머" 같은 검색은 회식·수업·회의 현장에서 매일 발생하는 상시 수요다.
- 기존 사이트들은 (a) 광고가 화면 절반, (b) 플래시 시절 유물 UI, (c) 모바일 미지원, (d) 빔프로젝터로 크게 띄우기 어려운 레이아웃이 많다.
- 100% 클라이언트 JS로 구현 가능 — 서버 필요 기능이 하나도 없다.

## 2. 목표 & 성공 지표

| 지표 | 3개월 | 6개월 |
|------|-------|-------|
| 월 방문자 | 5,000 | 30,000 |
| 도구별 상위 10위 키워드 | 3개 | 10개 |
| 재방문율 (도구 특성상 북마크형) | 20% | 30% |
| 월 AdSense 수익 | 승인 | 15만 원+ |

## 3. 타겟 사용자

- 교사·강사: 자리 배치, 발표자 뽑기, 팀 나누기, 수업 타이머 (빔프로젝터 화면)
- 직장인: 점심 메뉴 룰렛, 사다리타기(커피 내기), 회의 타임박스
- 사용 맥락: **즉시성** — 검색 → 3초 안에 사용 가능해야 함. 회원가입·설정 강요 시 즉시 이탈

## 4. 도구 라인업 (출시 순서 = 검색량 순)

### Phase 1 (런칭 필수)
1. **사다리타기** — 참가자/결과 입력, 애니메이션, 결과 공유 URL(파라미터 인코딩)
2. **원판 룰렛** — 항목 입력 → 돌리기. "점심 메뉴" 프리셋 내장
3. **팀 나누기 / 제비뽑기** — 명단 붙여넣기 → N팀 랜덤 분배
4. **대형 타이머/스톱워치** — 전체화면, 빔프로젝터 대응, 종료 알림음

### Phase 2
5. 자리 바꾸기 (교실 배치 그리드)
6. 발표 순서 뽑기
7. 숫자 뽑기 (범위 랜덤)
8. 주사위 / 동전 던지기

### Phase 3 (검색 유입 확장)
9. 글자수 세기 (자소서 시즌 검색량 大)
10. 맞춤법 간이 검사 링크 허브 / D-Day 계산기

**공통 설계 원칙**
- 입력값 localStorage 자동 저장 (재방문 시 명단 유지 → 교사 재방문 유도)
- 결과 URL 공유 (`?data=base64...`) — 서버 없이 상태 공유
- 전체화면 모드 (프로젝터), 다크모드
- 광고는 도구 영역 밖에만 — 기존 경쟁자의 최악 단점을 차별화 포인트로

## 5. 기술 스택

| 항목 | 선택 | 이유 |

## 6. 페이지 구조 & SEO

```
/                 도구 전체 그리드 (허브)
/ladder           사다리타기      ← "사다리타기" 월 수만 검색
/roulette         룰렛            ← "룰렛 돌리기", "점심 메뉴 룰렛"
/team             팀 나누기
/timer            타이머          ← "타이머 10분" 류 롱테일
/timer/10min      프리셋 랜딩 페이지 (프로그래매틱: 1분~60분)
/seat             자리 바꾸기
...
```

- 도구당 1 URL + 하단에 사용법·FAQ 텍스트 300–500자 (씬 콘텐츠 판정 방지, `FAQPage` 스키마)
- 타이머 프리셋 페이지(`/timer/5min` 등)로 롱테일 흡수 — 정적 생성 수십 페이지
- 도구 간 내부 링크 ("사다리 대신 룰렛은 어때요?")

## 7. 수익화

1. AdSense: 헤더 아래·푸터 위 고정 슬롯. 도구 상호작용 영역 침범 금지
2. 쿠팡파트너스: 맥락 제휴 (타이머 페이지 → 구글 타임타이머, 교실 도구 → 문구류)

## 12. 핵심 원칙

1. **3초 사용성**: 검색 착지 → 첫 사용까지 3초. 설정 강요·회원가입·로딩 스피너 금지.
2. **광고 절제 설계**: 광고 슬롯은 도구 상호작용 영역 밖에만 (경쟁자의 최악 단점이 우리 차별화).
3. **공정한 랜덤**: 셔플·추첨은 Fisher-Yates. 사다리 결과는 반드시 순열(전단사) — 단위 테스트로 증명.
4. **프로젝터 대응**: 타이머·룰렛·사다리는 전체화면 모드 필수 (교실 빔프로젝터 사용 맥락).
5. **재방문 유도**: 입력한 명단은 localStorage 자동 저장 — 교사가 내일 또 와도 명단 그대로.
6. **서버 없는 공유**: 결과 공유는 URL 파라미터(base64) 인코딩.
7. **점진 개선**: 단계마다 시각·성능·느낌 +12%, 매 단계 브라우저 실검증.

## 14. 완성 기준 (Definition of Done)

- [ ] 허브(index) + 도구 4종: 사다리타기·룰렛·팀 나누기·타이머, 각각 콘솔 에러 0
- [ ] 사다리: 참가자 2~10명, 결과 항목 입력, 가로줄 랜덤 생성, 경로 애니메이션, 결과가 항상 순열(단위 테스트)
- [ ] 룰렛: 항목 2~12개, 프리셋(점심 메뉴) 내장, Canvas 회전 애니메이션 + 감속, 균등 확률
- [ ] 팀 나누기: 명단 줄단위 붙여넣기 → N팀 균등 분배(±1), Fisher-Yates 공정성 테스트
- [ ] 타이머: 분/초 설정 + 프리셋(1/3/5/10/30분), 대형 표시, 전체화면, 종료음(WebAudio), 일시정지/재개
- [ ] 명단·설정 localStorage 자동 저장/복원
- [ ] 사다리·팀나누기 결과 URL 공유 (파라미터 인코딩) + 클립보드 복사
- [ ] 다크/라이트 테마, 375px 모바일 + 데스크톱
- [ ] 각 도구 하단 사용법·FAQ 텍스트 (씬 콘텐츠 방지) + FAQPage 스키마
- [ ] SEO: 페이지별 title/메타/OG/canonical, robots, sitemap, llms.txt
- [ ] Node 단위 테스트 전부 통과 (셔플·순열·분배·시간 포맷)
- [ ] GitHub Pages 배포 + 전 리소스 200 확인
